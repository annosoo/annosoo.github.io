MongoDB安装配置启动停止

发表于 2018-04-23

折腾感悟:
对于一个新技术的学习最佳选择是参考官方文档，其次近期的教学视频。再次书籍。
对于出来有段时间的。优先书籍。

正文:
这是一个更新频繁的软件。
很多书上的指令在MongoDB v3.6用不了。
查看系统服务 run + services.msc（检查mongodb服务是否启动）

简言之
1、下载解压安装
2、exe所在文件夹添加到系统环境变量。
3、创建数据存储文件夹 data\db，日志文件夹 log（需要手动创建）
下面都是两个短横，没显示完全
run cmd as admin
mongod –directoryperdb –dbpath C:\mongodb\data\db –logpath C:\mongodb\log\mongo.log –logappend –install
4、net start MongoDB启动服务
5、net stop MongoDB关闭服务

hexo常用指令

发表于 2018-04-22

HEXO 用得最频繁的几个指令（参阅他人博客）

hexo s (hexo server的缩写)
启动本地服务器,用于预览主题.默认地址:localhost:4000预览同时可以修改文章内容,保存后刷新页面对配置文件_config.yml的修改，需要重启本地服务器

hexo n “标题”(hexo new)
新建标题文章

hexo d（hexo deploy）
自动生成网站静态文件，部署到设定的仓库

hexo clean
清除缓存文件db.json和已生成的静态文件public。网站异常是可执行试试。

hexo g（hexo generate）
生成静态文件到默认设置的public文件夹。如果自动部署，不需先执行该指令

常用组合
hexo clean && hexo g && hexo d

hexo遇到的问题汇总

发表于 2018-04-22

1、 .md文件的注释
不能识别 //

2、 typeErrror: can not read “offset” of
nulloffset时间偏移的意思。问题原因出在配置文件里的timezone变量。一开始在timezone 后面加了 //以为注释方法是通用的。 试了很多次一直报错。尝试网上的给办法，给timezone赋一个具体的时区，并将hexo文件下，和主题文件夹下配置里的timezone改成一致。后来测试发现都不是问题的关键。timezone本来不处理，取的是系统时区。将 后面的 //去掉，就可以了。
